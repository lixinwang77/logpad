# Logpad 版本历史

## 版本规范

采用语义化版本 `MAJOR.MINOR.PATCH`（例如 `1.2.3`）：

| 变更类型 | 版本号变化 | 示例 |
|----------|------------|------|
| Bug 修复 | PATCH +1 | 1.0.0 → 1.0.1 |
| 新功能（向后兼容） | MINOR +1，PATCH 归零 | 1.0.1 → 1.1.0 |
| 不兼容的重大变更 | MAJOR +1，MINOR/PATCH 归零 | 1.1.0 → 2.0.0 |

每次发版还需递增 **Build 号**（`CURRENT_PROJECT_VERSION`），与版本号同步更新。

### 发版步骤

1. 修改 `Logpad/Logpad.xcodeproj/project.pbxproj` 中的 `MARKETING_VERSION` 和 `CURRENT_PROJECT_VERSION`（Debug / Release 两处都要改）
2. 在本文件「变更记录」中追加条目
3. 提交代码

## 变更记录

### 1.16.1 (2026-06-11)

- 调整预设过滤词的点击行为：点击过滤词文字改为切换该词的勾选/取消勾选（与点勾选框等效），不再直接把词加入搜索框；套用统一通过点击组名进行（带入已勾选的词）。移除单词直接套用相关逻辑

### 1.16.0 (2026-06-11)

- 预设侧栏新增过滤词启用开关：每个过滤词左侧加勾选框，仅勾选的词在「套用整组」时生效，未勾选的词被排除（文字置灰）；点击单词直接套用不受影响。新添加的词默认勾选，启用状态随预设持久化（`FilterPresetWord` 新增 `isEnabled` 字段，向后兼容旧数据默认勾选）
- 「套用整组」改为对本组词做同步：先移除搜索框里属于本组的词，再加入当前已勾选的词，因此取消勾选某词后再次套用会从搜索框去掉该词；其它来源（手输/别的组）的词保持不动

### 1.15.1 (2026-06-11)

- 修复预设侧栏重复套用同一组/词时，过滤词被重复追加到搜索框的问题（如点两次同一组得到 `a|b|a|b`）：`applyPreset` 现按 `|` 拆分现有关键词去重，只追加尚不存在的词，重复点击不再叠加

### 1.15.0 (2026-06-11)

- 新增搜索预设侧栏（重新定义 M6）：窗口左侧新增可显隐（每窗口独立）、可拖宽的预设侧栏，按「组（带组名）→ 多个过滤词」管理常用过滤词。点击组名套用组内全部词（`|` 合并）、点击单词套用单词，均以 `|` 追加到搜索框并自动开启 Regex，触发搜索与自动分屏；增删改组/词全部内联在侧栏完成（删组带确认；组名全局唯一、不区分大小写，重名时提示；内联输入框失焦即提交收起）。组名左侧箭头可折叠/展开组内过滤词，折叠状态持久化。鼠标悬停整组任意位置即显示操作图标（整块矩形可 hover）。预设由单例 `PresetStore` 持有，写入 `UserDefaults`（`filterPresetGroups`），全窗口/标签页共享并随重启保留。新增 `PresetStore.swift`、`PresetSidebarView.swift`，`Models.swift` 新增 `FilterPresetGroup` / `FilterPresetWord`

### 1.14.0 (2026-06-10)

- 新增超大文件索引进度条：打开文件 ≥ 20MB 时，主视图显示确定性进度条（`LoadingView`），按「已扫描字节 / 文件总大小」实时展示百分比与文件大小，节流约每前进 1% 刷新一次，避免多 GB 文件刷屏；小文件秒级完成不显示进度条以免闪烁。`FileReader` 新增 `fileSize` 与 `indexProgress` 状态
